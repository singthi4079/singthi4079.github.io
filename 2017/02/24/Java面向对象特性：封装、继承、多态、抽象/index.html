Java面向对象特性：封装、继承、多态

封装

封装的定义

在面向对象程式设计方法中，封装（英语：Encapsulation）是指，一种将抽象性函式接口的实作细节部份包装、隐藏起来的方法。

封装有什么作用

  1. 良好的封装能够减少耦合;
  2. 类内部的结构可以自由修改;
  3. 可以对成员变量进行更精确的控制;
  4. 隐藏信息，实现细节.

  1. 使用访问限定修饰符修饰类的实例变量或方法

继承

继承的定义

继承就是子类继承父类的特征和行为，使得子类对象（实例）具有父类的实例域和方法，或子类从父类继承方法，使得子类具有父类相同的行为。

继承有什么作用

  1. 服用以前的代码, 提高开发效率
  2. 让类与类之间产生了关系，是多态的前提

每个人都会有身高、体重、年龄、性别等属性, 而男人与女人之间不是完全一样的. 所以在设计的时候可以将人抽象为一个类, 存放男人和女人公共的属性, 再定一个一个男人类和一个女人类, 继承人这个类.

Man继承Person类后, Man类就拥有Person类的属性、方法; 在Man类中还可以定义自己独有的实例变量和方法.

同时在使用继承时需要记住三句话：

  1. 子类拥有父类非private的属性和方法。
  2. 子类可以拥有自己属性和方法，即子类可以对父类进行扩展。
  3. 子类可以用自己的方式实现父类的方法。

多态

定义

多态就是指程序中定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用在编程时并不确定，而是在程序运行期间才确定，即一个引用变量倒底会指向哪个类的实例对象，该引用变量发出的方法调用到底是哪个类中实现的方法，必须在由程序运行期间才能决定。因为在程序运行时才确定具体的类，这样，不用修改源程序代码，就可以让引用变量绑定到各种不同的类实现上，从而导致该引用调用的具体方法随之改变，即不修改程序代码就可以改变程序运行时所绑定的具体代码，让程序可以选择多个运行状态，这就是多态性。

简单说就是父类对象引用子类对象. B类继承A类.
