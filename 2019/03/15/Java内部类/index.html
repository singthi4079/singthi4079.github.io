内部类是指在一个外部类内部再定义一个类. 内部类作为外部类的一个成员, 依附于外部类存在.

内部类可以是静态的, 可以用protected和private修饰. 内部类有以下几类: 成员内部类, 局部内部类, 静态内部类, 匿名内部类.

内部类的共性:

  1. 内部类也是一个独立的类, 在编译后内部类被编译成独立的.class文件.
  2. 内部类不能用普通的方式访问
  3. 内部类被声明为静态的, 就不能访问外部类的成员变量, 只能访问外部类的静态成员变量
  4. 外部类不能直接访问内部类的成员, 需要通过内部类对象访问

内部类可以自由的访问外部类的成员变量, 不管是否是private的

因为当外部类的对象创建内部类对象时, 内部类会得到一个外部类对象的引用, 通过这个引用, 可以访问外部类对象的全部状态.

  1. 编译器自动为内部类创建一个外部类的成员变量, 这个变量指向外部类对象的.
  2. 编译器自动为内部类的构造方法添加一个外部类类型的参数. 并在构造方法中为外部类类型的成员变量赋值.
  3. 调用内部类的构造函数初始化内部类对象时, 默认传入外部类的引用.

使用内部类的原因

  • 内部类方法可以访问外部类定义的所有成员变量, 包括私有成员变量

  • 对同一个包中其他类隐藏

  • 实现多重继承

  • 回调函数(匿名内部类比较便捷)

成员内部类

在一个类中直接定义内部类, 成员内部类和普通成员没什么区别. 成员内部类不能含有static的变量和方法

局部内部类

在方法中定义的类称为局部内部类. 和局部变量类似, 只能访问当前代码块内的常量, 和外部类的所有成员.

  1. 局部内部类只能在定义该内部类的方法实例化, 不能在此方法外对其实例化
  2. 局部内部类对象不能使用该内部类所在方法的非final局部变量

静态内部类

普通内部类对象隐含一个指向外部类对象的引用. 而静态内部类不是这样的

  1. 不需要外部类的对象就可以创建静态内部类的对象
  2. 不能从静态内部类的对象访问非静态的外部类对象

匿名内部类

什么时候需要使用匿名内部类:

  • 只用到类的一个实例
  • 类定义后马上用刀
  • 类非常小
  • 给类命名并不会导致你的代码更容易被理解

原则:

  • 匿名内部类不能有构造方法
  • 不能定义任何静态成员变量, 方法, 类
  • 不能使用public, protected, private, static
  • 只能创建一个实例
  • 匿名内部类事局部内部类
