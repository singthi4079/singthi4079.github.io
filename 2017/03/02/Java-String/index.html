Java String

String 不是基本数据类型, 而是一个对象, 并且是不可变的对象. String 类被定义为final, 即不能被继承. 字符串对象在初始化之前, 它的值为null. “”, null, new String()三者的区别.

  1. null表示String 还没有new, 对象的引用还没有创建, 也没有分配内存空间
  2. “”, new String()说明已经new了一个对象. 只是内部为空

在进行String连接操作(+)时, 连接操作最开始时如果是字符串常量, 编译后会把尽可能多的字符串常量连接起来, 形成新的字符串常量与后续连接.

字符串连接是从左向右依次进行, 对于不同的字符串, 首先以最左边的字符串为参数创建StringBuilder对象, 然后依次对右边进行append操作. 最后把StringBuilder对象通过toString()方法转换成String对象. 中间存在多个字符串常量时不会自动拼接.

当使用+进行多个字符串连接时，实际上是产生了一个StringBuilder对象和一个String对象。

常用方法:

  1. 字符串比较
    • equals 判断内容是否相同
    • compareTo 判断字符串的大小关系
    • == 判断内容与地址是否相同
    • equalsIgnoreCase 忽略大小写的情况下判断内容是否相同
    • reagionMatches() 对字符串中的部分内容是否相同进行比较

JVM为了提高性能和减少内存开销, 内部维护一个字符串常量池, 每创建一个字符串常量, JVM首先检查字符串常量池, 如果存在, 返回此种字符串对象的引用, 不存在就创建该字符串对象放入池中.

这种方式与创建字符串常量的方式不同, 当使用new String(str) 方式创建字符串对象时, 不管字符串常量池中是否有与此相同内容的字符串, 都会在堆内存创建新的字符串对象.

intern()方法返回字符串对象在字符串常量池中的对象引用. 如果字符串常量池中没有这个字符串, 就创建一个新的字符串放到常量池中.

使用StringBuffer处理字符串时, 如果对字符串进行修改操作, 并不会产生一个新的字符串对象.

在修改时StringBuffer都是修改自身, 而String 类则是产生一个新的对象.

StringBuffer必须通过构造方法进行初始化.

StringBuilder也是一个可变的字符串对象, 和StringBuffer不同在于它是线程不安全的. 所以StringBuilder比StringBuffer速度快.

可以通过Java的反射机制修改String的内容
