Java集合接口

Java类库中集合类的基本接口是Collection接口. 这个接口有两个基本方法:

add方法用于向集合添加元素, 如果添加元素改变了集合返回true, 否则就返回false.

iterator方法返回一个实现了Iterator接口的对象. 用这个迭代器对象依次访问集合中的元素.

Iterator接口有三个方法:

反复调用next方法可以逐个访问集合中的每个元素. 如果到了集合的末尾, next方法会抛出一个NoSuchElementException. 在调用next方法前要先调用hasNext方法.

使用for each循环遍历集合

编译器会把for each循环翻译为带有迭代器的循环.

Java迭代器的查找操作与位置变更是紧密相连的. 查找一个元素的唯一方法是调用next.

执行查找操作的同时, 迭代器的位置也向前移动.

应该认为Java迭代器是位于两个元素之间. 当调用next时, 迭代器会越过下一个元素, 并返回刚才越过的那个元素的引用.

Iterator接口的remove方法会删除上次调用next方法时返回的元素.

要删除指定位置上的元素, 必须要迭代器越过这个元素.

在调用remove之前必须要调用next, 否则会抛出一个IllegalStateException异常.
