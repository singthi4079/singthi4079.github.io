Java-泛型

Veröffentlicht am 2017-03-04

泛型本质上是提供类型的”类型参数”, 也被称为参数化类型或参数多态. 使用泛型的Java程序和普通的Java程序基本相同, 只是多了参数化的类型同时少了一些类型转换.

实际上使用泛型的程序首先被转化为一般不带泛型的Java程序再进行处理. 编译器自动完成转化, 具体步骤:

  1. 把参数化类型的类型参数擦除

  2. 把类型变量用”上限”取代, 通常是Object. 类型变量指的是: 实例域, 本地方法域, 方法参数和方法返回值中用来标记类型信息的变量. 如：

    1. 实例域: A a;
    2. 方法声明: Node (A a){}

    A就是类型变量

  3. 添加类型转换并插入”桥方法”, 方便overridden能够正常工作

使用泛型的特点:

  1. 类型安全. 泛型的主要目标就是提高类型安全
  2. 消除强制类型转换.
  3. 向后兼容.
  4. 层次清晰, 恪守规范. 泛型主要是在编译器层面实现的, 对JVM是透明的
  5. 性能收益.

Java枚举-Enum

Veröffentlicht am 2017-03-03

如果需要描绘彩虹的七种颜色, Java程序中通常使用常量定义方式来实现.

使用的时候，你可以在程序中直接引用这些常量。但是，这种方式还是存在着一些问题。

  1. 类型不安全

    由于颜色常量的对应值是整数形，所以程序执行过程中很有可能给颜色变量传入一个任意的整数值，导致出现错误。

  2. 没有命名空间

    由于颜色常量只是类的属性，当你使用的时候不得不通过类来访问。

  3. 一致性差

    因为整形枚举属于编译期常量，所以编译过程完成后，所有客户端和服务器端引用的地方，会直接将整数值写入。这样，当你修改旧的枚举整数值后或者增加新的枚举值后，所有引用地方代码都需要重新编译，否则运行时刻就会出现错误。

  4. 类型无指意性

    由于颜色枚举值仅仅是一些无任何含义的整数值，如果在运行期调试时候，你就会发现日志中有很多魔术数字，但除了程序员本身，其他人很难明白其奥秘。

定义Enum类型语法:

enum 关键字加上名称和大括号包含起来的枚举值体即可

enum RainbowColor { RED, ORANGE, YELLOW, GREEN, CYAN, BLUE, PURPLE }

Enum 类型默认实现 Comparable 和 Serializable 接口.

Enum类型使用:

用 Enum 时候有几个地方需要注意:

  1. enum 类型不支持 public 和 protected 修饰符的构造方法，因此构造函数一定要是 private 或 friendly 的。也正因为如此，所以枚举对象是无法在程序中通过直接调用其构造方法来初始化的。
  2. 定义 enum 类型时候，如果是简单类型，那么最后一个枚举值后不用跟任何一个符号；但如果有定制方法，那么最后一个枚举值与后面代码要用分号’;’隔开，不能用逗号或空格。
  3. 由于 enum 类型的值实际上是通过运行期构造出对象来表示的，所以在 cluster 环境下，每个虚拟机都会构造出一个同义的枚举对象。因而在做比较操作时候就需要注意，如果直接通过使用等号 ( ‘ == ’ ) 操作符，这些看似一样的枚举值一定不相等，因为这不是同一个对象实例。

Java String

Veröffentlicht am 2017-03-02

String 不是基本数据类型, 而是一个对象, 并且是不可变的对象. String 类被定义为final, 即不能被继承. 字符串对象在初始化之前, 它的值为null. “”, null, new String()三者的区别.

  1. null表示String 还没有new, 对象的引用还没有创建, 也没有分配内存空间
  2. “”, new String()说明已经new了一个对象. 只是内部为空

在进行String连接操作(+)时, 连接操作最开始时如果是字符串常量, 编译后会把尽可能多的字符串常量连接起来, 形成新的字符串常量与后续连接.

字符串连接是从左向右依次进行, 对于不同的字符串, 首先以最左边的字符串为参数创建StringBuilder对象, 然后依次对右边进行append操作. 最后把StringBuilder对象通过toString()方法转换成String对象. 中间存在多个字符串常量时不会自动拼接.

当使用+进行多个字符串连接时，实际上是产生了一个StringBuilder对象和一个String对象。

常用方法:

  1. 字符串比较
    • equals 判断内容是否相同
    • compareTo 判断字符串的大小关系
    • == 判断内容与地址是否相同
    • equalsIgnoreCase 忽略大小写的情况下判断内容是否相同
    • reagionMatches() 对字符串中的部分内容是否相同进行比较

JVM为了提高性能和减少内存开销, 内部维护一个字符串常量池, 每创建一个字符串常量, JVM首先检查字符串常量池, 如果存在, 返回此种字符串对象的引用, 不存在就创建该字符串对象放入池中.

这种方式与创建字符串常量的方式不同, 当使用new String(str) 方式创建字符串对象时, 不管字符串常量池中是否有与此相同内容的字符串, 都会在堆内存创建新的字符串对象.

intern()方法返回字符串对象在字符串常量池中的对象引用. 如果字符串常量池中没有这个字符串, 就创建一个新的字符串放到常量池中.

使用StringBuffer处理字符串时, 如果对字符串进行修改操作, 并不会产生一个新的字符串对象.

在修改时StringBuffer都是修改自身, 而String 类则是产生一个新的对象.

StringBuffer必须通过构造方法进行初始化.

StringBuilder也是一个可变的字符串对象, 和StringBuffer不同在于它是线程不安全的. 所以StringBuilder比StringBuffer速度快.

可以通过Java的反射机制修改String的内容

Veröffentlicht am 2017-03-01

Java对象初始化

Java中一个对象在被使用前必须要被正确的初始化, 这是Java规范规定的.

Java对象何时被初始化

Java对象在被创建的时候初始化. Java中有两种行为会引起对象的创建.

  1. 显式对象创建: 通过new关键字调用一个类的构造函数.

  2. 隐式对象创建

    • 加载一个包含String字面量的类或者接口会引起一个新的String对象被创建. 如果包含相同字面量的String对象已经存在虚拟机内(JVM会在内存中为所有碰到的String字面量维护一个列表. 程序中使用相同字面量的都会指向同一个String对象)

    • 自动封装机制可能会引起一个原子类型的包装类对象被创建

    • String 连接符也可能引起新的String或者StringBuilder对象被创建. 同时还可能引起原子类型的包装对象呗创建.

Java如何初始化对象

当对象被创建后, JVM会为其分配内存, 用于存放对象实例变量和从父类继承过来的实例变量. 即使从父类继承过来的实例变量可能被隐藏也会分配空间. 在分配内存的同时, 实例变量会被富裕默认值.

在main方法中，我们用Foo引用一个Bar对象，如果实例变量与方法一样，允许被覆盖，那么打印的结果应该是1，但是实际的结果确是0。
但是如果我们在Bar的方法中直接使用i，那么用的会是Bar对象自己定义的实例变量i，这就是隐藏，Bar对象中的i把Foo对象中的i给隐藏了，这条规则对于静态变量同样适用。

内存分配完成后, 虚拟机会开始对新创建的对象执行初始化操作. Java中有三种执行对象初始化的结构. 分别是: 实例初始化器, 实例变量初始化器, 构造函数

  1. 构造函数

    每个类至少有一个构造函数, 如果没有显式定义构造函数, Java编译器会自动生成一个无参构造函数. 在生成的字节码中, 构造函数被命名为 方法

    在一个对象被初始化前, 它的超类必须被初始化. 除Object对象之外的所欲对象构造函数的第一条语句必须时父类的构造函数的调用语句, 或者事类中定义的其他构造函数.如果我们即没有调用其他的构造函数, 也没有显式调用超类的构造函数, 那么编译器会为我们自动生成一个对超类构造函数的调用指令.

  2. 实例变量初始化器和实例初始化器

    在定义实例变量时, 对变量进行赋值, 赋值语句就是实例变量初始化器.

    在构造函数执行之前会先完成这些初始化操作.

Java static

Veröffentlicht am 2017-03-01

Java static关键字

在程序中任何变量或代码都是在编译时由系统自动分配内存来存储的, 而静态就是指在编译后所分配的内存一直存在, 直到程序退出内存才会释放这个空间. 只要程序在运行, 这块内存就一直存在.

static表示全局 或 静态 的意思, 用来修饰成员变量和成员方法, 也可以形成静态的代码块. 被static修饰的成员变量或方法不依赖类特定的实例, 被类的所有实例共享. 只要类被加载, 虚拟机就会根据类名在运行时数据区的方法区内找到它们.

static修饰的成员变量和成员方法习惯上称为静态变量和静态方法，可以直接通过类名来访问，访问语法为:

类名.静态方法名(参数列表…)

类名.静态变量名

用static修饰的代码块表示静态代码块,当Java虚拟机(JVM)加载类时,就会执行该代码块.

static 变量

被static修饰的变量被称为 静态变量 或 类变量, 没有被static修饰的变量称为实例变量. 静态变量和实例变量的区别:

  1. JVM只为静态变量分配一次内存, 在加载类的过程中完成内存的分配
  2. 静态变量可以直接通过类名访问
  3. 实例变量依赖于类的实例,
  4. 每创建一个类的实例, 就会给实例变量分配一次内存

static 方法

  1. 静态方法可以直接通过类名调用
  2. 在静态方法中不能用this, super关键字
  3. 不能在静态方法中直接访问类的实例变量和实例方法
  4. 可以在静态方法中访问静态变量和静态方法
  5. static方法不依赖任何实例, 所以static方法必须被实现, 不能是abstract的

static代码块

static代码块是在类中独立于类成员的static语句块. 可以有多个, 位置可以随意放. 不在任何方法体内, JVM加载类时就会执行静态代码块. 如果一个类中有多个静态代码块, JVM会按照它们在类中出现的先后顺序依次执行. 每个静态代码块只会执行一次.

final关键字

Veröffentlicht am 2017-02-28

final关键字

如果希望程序中的某些数据不能改变, 可以把这些数据声明为final. final是Java的关键字, 表示这部分是无法修改的. 不希望被改变的原因有两个: 效率, 设计. 使用到 final 的有三种情况: 数据、方法、类。

数据

有时候数据的恒定不变是很有用的, 能够减轻系统运行时的负担. 这样的数据叫做常量. 常量主要用以下两种情况

  1. 编译期常量, 永远不能改变
  2. 运行期初始化时, 希望不会被改变

编译期常量在类加载的过程就已经完成初始化, 类加载完成后时不可更改的. 只能使用基本类型, 必须在定义时初始化.

运行期常量可以是基本数据类型, 也可以是引用数据类型. 基本数据类型的内容不能变, 引用数据类型的引用不能改变, 引用所指定的对象内容是可变的.

final方法

被final标注的方法都不能被继承, 重写(override). 使用final方法的原因:

  1. 方法锁定, 防止子类修改方法
  2. 提高效率

调用对象方法的执行过程

  1. 编译器查看对象的声明类型和方法名. 获取所有可能被调用的候选方法.

    假设调用x.f(param), 且隐式参数x声明为C类的对象. 可能存在多个名字为f, 但参数类型不一样的方法. 编译器会一一列举所有C类中名为f的方法和其超类中的访问属性为public且名为f的方法. 超类的私有方法不能访问.

  2. 编译器查看调用方法时提供的参数类型. 获得需要调用的方法名和参数类型.

    如果存在一个与提供的参数类型完全匹配, 就会选择这个方法. 这个过程被称为重载解析. 例如: x.f("hello"), 编译器会选择f(String), 而不是f(int). 由于允许类型转换(int -> double, B -> A等). 如果编译器没有找到与参数类型匹配的方法, 或者发现经过类型转换后有多个方法匹配, 就会报错.

  3. 如果是private、static、final方法或构造器, 编译器可以准确的知道该调用哪个方法, 这种方式称为静态绑定. 与之对应, 调用的方法依赖于隐式参数的实际类型, 并且在运行时实现动态绑定

  4. 当程序运行, 并且采用动态绑定调用方法时, 虚拟机一定调用与x所引用对象的实际类型最合适的那个类的方法.s

    假设x的实际类型是D, 它是C类的子类. 如果D类定义了方法f(String), 就直接调用它; 否则, 在D类的父类中寻找f(String).

每次调用方法都要进行搜索, 时间开销大. 虚拟机预先为每个类创建了一个方法表, 其中列出所有方法的签名和实际调用的方法. 在真正调用方法的时候, 虚拟机仅查找这个表就可以了.

动态绑定有一个重要的特性: 不需要对现有的代码进行修改, 就可以对程序进行扩展.

覆盖方法时, 子类方法不能低于父类方法的可见性. 如果父类方法是public, 子类方法一定要声明为public.

final 类

使用final修饰的类, 表明这是最终类, 不能被其他类继承.

对于 final 修饰的类来说，它的成员变量可以为 final，也可以为非 final。如果定义为 final，那么 final 数据的规则同样适合它。而它的方法则会自动的加上 final，因为 final 类是无法被继承，所以这个是默认的。

final参数

若某个参数被 final 修饰了, 则代表了该参数是不可改变的.

final 修饰参数在内部类中是非常有用的，在匿名内部类中，为了保持参数的一致性，若所在的方法的形参需要被内部类里面使用时，该形参必须为 final。

final 与 static

final 和 static 同时使用时即可修饰成员变量，也可修饰成员方法。

对于成员变量，该变量一旦赋值就不能改变，我们称它为”全局常量”。可以通过类名直接访问。

对于成员方法，则是不可继承和改变。可以通过类名直接访问。

动态绑定有一个重要的特性: 不需要对现有的代码进行修改, 就可以对程序进行扩展.

覆盖方法时, 子类方法不能低于父类方法的可见性. 如果父类方法是public, 子类方法一定要声明为public.

使用final修饰类, 这个类不能被继承.

使用final修饰方法, 在子类不能覆盖着歌方法. final类中的所有方法都是final的.

使用final修饰域, 构造对象后不允许修改它们的值.

抽象类不能被实例化, 即不能创建抽象类的实例.

任何声明为private的域或方法对其他类都是不可见的, 即使是子类也不能访问父类中的私有域和私有方法.

如果希望父类中的某些方法或某些域可以被子类访问, 可以把这些方法或域声明为protected

声明为protected的域或方法, 对所有子类及同一个包中所有其他类都是可见的.

4种访问修饰符:

  1. private: 仅本类可见;
  2. 默认(不需要修饰符): 对本包可见;
  3. protected: 对本包和所有子类可见;
  4. public: 对所有类可见

Object类中的equals方法用于检测一个对象是否等于另一个对象. 判断两个对象是否具有相同的引用.

Objects.equals方法: Objects.equals(a, b)如果两个参数都是null, 返回true, 如果只有一个为null, 返回false.

在子类中定义equals方法, 首先调用父类中的equals方法. 如果检测失败, 对象就不可能相等.

Java语言规范要求equals方法具有下面特性:

  1. 自反性: 对于任何非空引用x, x.equals(x)应该返回true
  2. 对称性: 对于任何引用x和y, 当且仅当y.equals(x)返回true, x.equals(y)也应该返回true
  3. 传递性: 对于任何引用x,y,z, 如果x.equals(y)返回true, y.equals(z)返回true, x.equals(z)也应该返回true
  4. 一致性:如果x和y引用的对象没有发生任何变化, 反复调用x.equals(y)应该返回同样的结果
  5. 对于任意非空引用x, x.equals(null)应该返回false

编写一个完美的equals方法的建议:

  1. 显式参数命名为otherObject, 稍后需要把它转换为另一个叫做other的变量

  2. 检测this和 otherObject是否引用同一个对象

    if (this == otherObject) return true;

  3. 检测otherObject是否为null, 如果为null, 返回false. 这个检测是必需的.

    if (otherObject == null) return false;

  4. 比较this和otherObject是否属于同一个类. 如果equals的语义在每个子类中有所改变, 就用getCass检测

    if (getClass() != otherObject.getClass()) return false;

    如果所有子类都拥有统一的语义, 就使用instanceof检测:

    if (!(otherObject instanceof ClassName)) return false;

  5. 将otherObject转换为相应的类类型变量

    ClassName other = (ClassName) otherObject;

  6. 现在开始对所有需要比较的域进行比较.

散列码(hash code)是由对象导出的一个整型值. 散列码是没有规律的. 如果x和y是两个不同的对象, x.hashCode()与y.hashCode()基本上不会相通.

hashCode方法定义在Object类中, 每个对象都有一个默认的散列码, 其值为对象的存储地址.

封装

封装的定义

在面向对象程式设计方法中，封装（英语：Encapsulation）是指，一种将抽象性函式接口的实作细节部份包装、隐藏起来的方法。

封装有什么作用

  1. 良好的封装能够减少耦合;
  2. 类内部的结构可以自由修改;
  3. 可以对成员变量进行更精确的控制;
  4. 隐藏信息，实现细节.

继承

继承的定义

继承就是子类继承父类的特征和行为，使得子类对象（实例）具有父类的实例域和方法，或子类从父类继承方法，使得子类具有父类相同的行为。

继承有什么作用

  1. 服用以前的代码, 提高开发效率
  2. 让类与类之间产生了关系，是多态的前提

每个人都会有身高、体重、年龄、性别等属性, 而男人与女人之间不是完全一样的. 所以在设计的时候可以将人抽象为一个类, 存放男人和女人公共的属性, 再定一个一个男人类和一个女人类, 继承人这个类.

Man继承Person类后, Man类就拥有Person类的属性、方法; 在Man类中还可以定义自己独有的实例变量和方法.

同时在使用继承时需要记住三句话：

  1. 子类拥有父类非private的属性和方法。
  2. 子类可以拥有自己属性和方法，即子类可以对父类进行扩展。
  3. 子类可以用自己的方式实现父类的方法。

多态

定义

多态就是指程序中定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用在编程时并不确定，而是在程序运行期间才确定，即一个引用变量倒底会指向哪个类的实例对象，该引用变量发出的方法调用到底是哪个类中实现的方法，必须在由程序运行期间才能决定。因为在程序运行时才确定具体的类，这样，不用修改源程序代码，就可以让引用变量绑定到各种不同的类实现上，从而导致该引用调用的具体方法随之改变，即不修改程序代码就可以改变程序运行时所绑定的具体代码，让程序可以选择多个运行状态，这就是多态性。

A a = new B();

Java基础-对象与类

类是构造对象的木板或蓝图. 由类构造对象的过程称为创建类的实例.

对象的三个主要特性:

  • 对象的行为 — 可以对对象施加哪些操作, 或可以对对象施加哪些方法
  • 对象的状态 — 当施加那些方法时, 对象如何响应
  • 对象标识 — 如何辨别具有相同行为与状态的不同对象

类之间的关系

  • 依赖(uses-a)
  • 聚合(has-a)
  • 继承(is-a)

依赖是uses-a关系, A类需要访问B类中的某个属性或方法. 一个类的方法操纵另一个类的对象. 应该尽可能减少类的相互依赖.

聚合是has-a关系, 类A的对象包含类B的对象.

继承是is-a关系, 是一种表示特殊与一般的关系.

构造器的名字与类名相同.

一个对象变量并没有包含一个对象, 而是引用对象.

Date d = new Date();

这条语句由两个部分

  1. new Date()构造了一个Date类型的对象
  2. 把新创建的Date对象的引用存储到变量d中

局部变量不会自动初始化, 只有实例变量才会自动初始化.

public类的名字必须与文件名相同, 在一个源文件中, 只能有一个公共类, 可以有任意个非公共类.

数据类型

Java中一共有8种基本类型

  1. 4种整型(byte, short, int, long)
  2. 2种浮点型(float, double)
  3. 1种表示Unicode编码的字符单元的字符类型(char)
  4. 1中表示真值的类型(boolean)

整型

整型表示没有小数部分的数值, 允许是负数. Java提供4种整型.

类型 存储需求 取值范围
byte 1字节 -128 ~ 127
short 2字节 -32768 ~ 32787
int 4字节 -2^32 ~ 2^32 -1
long 8字节 -2^64 ~ 2^64 -1

长整型数值有一个后缀L(40000L)

十六进制数值有一个前缀0x(0xCAFE)

八进制数值前缀0(010)

从Java 7开始, 加上前缀0b表示二进制数(0b1001)

Java 没有任何无符号类型(unsigned)

浮点类型

浮点类型用于表示有小数部分的数值.

类型 存储需求 取值范围
float 4字节 大约正负3.40282347E+38F(有效位数为6 ~ 7 位)
double 8字节 大约正负1.797693134862315E+308(有效位数为15位)

float 类型的数值有一个后缀F(3.14F), 没有后缀F的浮点数值(3.14)默认为double类型

表示溢出和出错情况的三个特殊的浮点数值:

  • 正无穷大(正整数除以0结果为无穷大) Double.POSITIVE_INFINITY
  • 负无穷大 Double.NEGATIVE_INFINITY
  • NaN(0/0 或 负数平方根 不是一个数字) Double.NaN

char 类型

char类型用于表示单个字符, 常用来表示字符常量. ‘A’是编码为65所对应的字符常量. 与”A”不通, “A”是一个包含字符A的字符串.

在Java中, char类型用UTF-16编码描述一个代码单元.

boolean 类型

boolean类型有两个值: false和true. 整型值和布尔值之间不能进行相互转换.

变量

Java中, 每个变量属于一种类型. 声明变量时, 变量的类型位于变量名之前.

每条语句必须以分号结束

变量名必须是一个以字母开头的由字母或数字构成的序列；

变量名大小写敏感, 长度没有限制

$不能在代码中使用这个字符, 它只用在Java编译器或其它工具生成的名字中.

声明一个变量后, 必须用赋值语句初始化. 不能使用未初始化的变量

赋值: int var=12;. 可以在任何地方声明变量, 尽量在变量第一次使用的地方声明.

常量

利用final关键字表示常量.

关键字final表示这个变量只能被赋值一次. 不能在更改. 通常常量名使用全大写.

运算符

除法: 当除数与被除数都是整数时, 表示整数除法; 否则, 表示浮点出发.

整数被0除会产生一个异常. 浮点数被0除会得到无穷大或者NaN

自增或自减运算符改变了变量的值, 它们的操作数不能是数值. (4++是非法的语句)

++n 先进行加1运算

n++则使用变量原来的值

三元操作符: condition ? e1:e2

数值类型之间的转换

安全转换: byte -> short -> int -> long; char -> int; int -> double;

不安全转换: int -> float; long -> float; long -> double;

两个类型不一样的数进行二元操作时, 其中一个数的类型的取值范围小于另一个数, 则取值范围较小的那个数会转换为取值范围较大的类型.

  • 如果两个操作数中有一个是double类型, 另一个操作数就会转为double类型
