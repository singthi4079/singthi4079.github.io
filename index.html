Java-线程同步与锁

Posté le 2017-03-22

线程同步是为了防止多个线程访问一个数据对象时, 对数据造成破环

例如: 两个线程A, B都操作同一个对象Foo对象, 并修改Foo对象上的数据

要保持数据合理性, 就要对Foo的访问加以限制, 每次只能一个线程访问:

  1. 吧竞争资源类Foo变量x 标识为private
  2. 同步修改变量的代码, 使用synchronized同步方法或代码

同步和锁定

每个对象都有一个内置锁.

当程序运行到非静态的synchronized同步方法时, 自动获得当前实例(this)有关的锁. 当程序运行到synchronized同步方法或代码时, 对象锁才起作用.

一个对象只有一个锁. 如果一个线程获得该锁, 其他线程就不能再获得锁, 直到第一个线程释放锁.

只能同步方法, 不能同步变量和类.

类可有同时拥有同步和非同步方法

线程睡眠时, 它所持的任何锁都不会释放

线程可以获取多个锁, 在一个对象的同步方法中调用另一个对象的同步方法

同步损害并发行.

使用同步代码块时, 应该指定在哪个对象上同步

同步静态方法时, 需要一个用于整改类对象的锁, 这个对象就是这个类(XXX.class)

如果线程试图进入同步方法, 而锁已被占用, 则线程在该对象上被阻塞. 实际上, 线程进入该对象的一种池中, 必须在那里等待, 知道锁被释放.

什么时候需要同步

在多个线程同时访问互斥（可交换）数据时，应该同步以保护数据，确保两个线程不会同时修改更改它。

对于非静态字段中可更改的数据，通常使用非静态方法访问。

对于静态字段中可更改的数据，通常使用静态方法访问。

如果需要在非静态方法中使用静态字段，或者在静态字段中调用非静态方法，问题将变得非常复杂

现在的操作系统是对任务操作系统, 多线程是实现多任务的一种方式

进程是指一个内存中运行的应用程序, 每个进程都有自己独立的一块内存空间.一个进程中可以启动多个线程.

线程是指进程中的一个执行流程, 一个进程可运行多个线程. 线程总是属于某个进程, 进程中的多个线程共享进程的内存.

Java中, “线程”指两件事:

  1. java.lang.Thread类的一个实例
  2. 线程执行

main()方法运行在主线程内.

一旦创建一个新的线程, 就产生一个新的调用栈

线程分两类: 用户线程, 守候线程

守候线程一般由操作系统或用户自己创建.

Java-TreeMap

Posté le 2017-03-19

红黑树是一颗二叉树, 具备二叉树的所有特性, 而且还是一颗自平衡的二叉树.

红黑树就是节点是红色或者黑色的平衡二叉树. 必须遵循一下规则:

  1. 每个节点都只能是红色或者黑色
  2. 根节点是黑色
  3. 每个叶节点(NIL节点, 空节点)都是黑色
  4. 如果一个节点是红色, 则它两个字节点都是黑色, 一条路径上不能出现相邻的两个红色节点
  5. 从任一节点到其每个叶子的所有路径都包含相同数目的黑色节点

TreeMap定义

TreeMap继承AbstractMap, 实现NavigableMap, Cloneable, Serializable接口. NavigableMap意味着它支持一系列的导航方法, 具备针对给定搜索目标返回最接近匹配项的导航方法.

对于叶子节点Entry是TreeMap的内部类, 它有几个重要的属性:

put方法

对于新节点插入有以下三个关键点:

  1. 插入新节点总是红色节点
  2. 如果插入节点的父节点是黑色, 能维持性质
  3. 如果插入节点的父节点是红色, 破坏了性质, 古插入算法就算通过重新着色或旋转, 来维持性质

新增节点过程:

  1. 根节点

    如果新插入的节点N没有父节点, 直接当作根节点插入即可, 同时把颜色设置为黑色

  2. 父节点为黑色

    这种情况新节点N同样直接插入, 同时颜色为红色. 根据规则四它会存在两个黑色的叶子节点, 值为null. 同时由于新增节点N是红色, 所以通过它的字节点的路径依然会保存着相同的黑色节点数, 同样满足规则5

  3. 如果父节点P或P的兄弟节点U都是红色

    这种情况直接插入会出现不平衡现象. P, U节点变黑, G节点变红. 这是由于经过节点P,U的路径都必须经过G所以这些路径上的黑色节点数还是相同的. 但是G节点的父节点也是红色, 这时需要将G节点当做新节点递归处理

  4. 若父节点P是红色, 叔父节点U是黑色或者缺失, 且新增节点N为P节点的右孩子

    这种情况下对新增节点N,P进行左旋. 这里产生的结果其实并没有完成, 还不是平衡的

  5. 若父节点P是红色, 叔父节点U是黑色或者缺失, 且新增节点N为P节点的左孩子

    这种情况可能是因为4产生的, 也可能不是. 对于这种情况先已P节点为中心进行右旋转, 在旋转后产生的树种, 节点P是节点N, G的父节点. 违反了规则4, 所以将P,G节点的颜色进行交换. 使之满足规范.开始时所有的路径都需要经过G其他们的黑色节点数一样，但是现在所有的路径改为经过P，且P为整棵树的唯一黑色节点，所以调整后的树同样满足规范5。

put方法的实现主要分为两个步骤:

  1. 构建排序二叉树
  2. 平衡二叉树

对于排序二叉树的创建, 添加节点的过程如下:

  1. 以根节点为初始节点进行检索
  2. 与当前节点进行比对, 若果新增节点较大, 则以当前节点的右节点作为新的当前节点. 否则当前节点的左子节点作为新的当前节点
  3. 循环递归2步骤, 直掉知道检索出合适的叶子节点为止
  4. 将新增节点与3步骤中找到的节点比对, 如果新节点较大, 则添加右子节点, 否则添加为左子节点

进行修复工作:调整的过程务必会涉及到红黑树的左旋、右旋、着色三个基本操作

左旋: rotateLeft()

所谓左旋转，就是将新增节点（N）当做其父节点（P），将其父节点P当做新增节点（N）的左子节点。即：G.left —> N ,N.left —> P。

右旋：rotateRight()

所谓右旋转即，P.right —> G、G.parent —> P

Java-HashTable

Posté le 2017-03-15

HashTable的定义:

HashTable继承Dictionary类, 实现Map接口, Dictionary类是任何可将键映射到相应值的类的抽象父类. 任何一个Dictionary对象中, 每个键至多与一个值相关联.

HashTable采用”拉链法”实现哈希表, 定义有几个重要参数: table, count, threshold, loadFactor, modCount.

  • table: 为一个Entry[]数组类型, Entry表示”拉链”的节点, 每个Entry表示一个键值对, 哈希表的”key-value键值对”, 都是存储在Entry数组中
  • count: HashTable的大小, 指容器中包含Entry键值对的数量
  • threshold: HashTable的阈值, 用于判断是否需要调整HashTable的容量. threshold＝容量*负载因子
  • loadFactory: 负载因子
  • modCount: 用于实现”fail-fast”机制(快速失败).

快速失败是在并发集合中, 其进行迭代操作时, 如果其他线程对其进行结构性的修改, 这时迭代器立马感知到, 并抛出ConcurrentModificationException异常.

构造方法

默认构造容量为11, 负载因子为0.75

指定初始化容量和默认负载因子

initHashSeedAsNeeded方法用于初始化hashSeed参数, 其中hashSeed用于计算key的hash值, 它与key的hashCode进行按位异或运算.

hashSeed是一个与实例相关的随机值, 主要用来解决hash冲突.

构造一个与给定的 Map 具有相同映射关系的新哈希表

put方法

put方法将指定key映射到此哈希表中的指定value. 这里的key和value都不能为空

  1. 计算key的hash值
  2. 根据hash值获得key在table数组中的索引位置
  3. 迭代key处的Entry链表, 如果链表存在一个相同的key对象, 直接替换value
  4. 不存在相同的key, 把key-value节点插入到改index索引位置

  1. 在put方法中, 如果要向table[]中添加元素, 会先进行容量的校验. 如果容量达到阈值, 会进行扩容处理

>

  1. 计算索引时hash & 0x7FFFFFFF这一步是为了把负数转换为正数,

HashTable与HashMap的区别

  1. HashTable基于Dictionary类, HashMap基于AbstractMap.
  2. HashMap可以允许存在一个null的key和任意个null的value, 但HashTable的key和value都不允许为null
  3. HashTable的方法是同步的, 而HashMap的方法不是同步的.

Collections类存在一个静态方法: synchronizedMap()这个方法创建了一个线程安全的Map对象, 并把它作为一个封装对象来返回. 所以通过Collections类的synchronizedMap方法是可以我们你同步访问潜在的HashMap

Java-HashMap

Posté le 2017-03-14

HashMap是基于哈希表的Map接口的实现, 以key-value的形式存在. 在HashMap中, key-value总是作为一个整体处理. 系统根据hash算法来计算key-value的存储位置

定义

HashMap实现了Map接口, 继承AbstractMap. Map接口定义了key映射到value的规则.

构造函数

HashMap有三个构造函数:

  • HashMap(): 构造一个具有默认初始容量(16)和默认加载因子(0.75)的空HashMap
  • HashMap(int initialCapacity): 构造一个带有指定初始容量和默认加载因子的空HashMap
  • HashMap(int initialCapacity, float loadFactor): 构造一个指定初始容量和加载因子的空HashMap

容量表示哈希表中桶的数量

加载因子是哈希表在其容量自动增加之前可以达到多满的一种尺度. 它衡量的是一个散列表的空间使用程度, 负载因子越大表示散列表装填程度越高.

对于使用链表的散列表来说, 查找一个元素的平均时间是O(1+a), 因此如果负载因子越大, 对空间的利用率越充分, 但是后果是查找效率越低.

负载因子太小, 因为散列表数据过于稀疏, 对空间造成严重的浪费.

HashMap指出快速存储.

数据结构

Java最常用的两种结构是数组和引用, 几乎所有的数据结构都可以利用这两种来组合实现

HashMap实际上是一个链表散列, 数据结构如下:

上图看出HashMap底层有用数组实现, 数组中每一项都是一条链. initialCapacity表示数组的长度

每次新建一个HashMap时, 都会初始化一个table数组, 元素为Entry节点

HashMap的put方法

HashMap保存数据的过程

  1. 判断key是否为null, 如果是null, 直接调用putForNullKey方法
  2. 如果key不是null, 先计算key的hash值
  3. 根据hash值搜索在table数组中的索引位置
  4. 如果table数组在该位置处有元素, 通过比较是否存在相同的key, 如果存在就覆盖key的value, 否则就保存元素到链头(最先保存的元素放在链尾)
  5. 如果没有元素直接爆粗

注意点:

  1. 迭代的地方: 迭代的原因是为了防止存在相同的key, 如果两个key的hash值相同, HashMap的处理方式是用新的value替换旧value
  2. (1), (2)处, hash方法是一个纯粹的数学计算, 用于计算hash值

HashMap使用indexFor方法保证table元素分布均匀

HashMap的底层数组长度总是2的n次方(capacity <<= 1;), 当length为2的n次方时, h&(length-1)相当于对length取模, 而怯速度比直接取模快得多.

h&(length – 1)除了取模运算外, 还负责均匀分布table数据和充分利用空间

假设length=16(2^n) 和 15, h=5,6,7

当n=15时, 6和7的结果一样, 表示他们在table的存储位置相同, 产生了碰撞, 6,7会在一个位置形成链表, 导致查询速度下降. 看看0-15的结果:

表中看出总共发生了8次碰撞, 同时浪费空间非常大

而当length = 16时，length – 1 = 15 即1111，那么进行低位&运算时，值总是与原来hash值相同，而进行高位运算时，其值等于其低位值。所以说当length = 2^n时，不同的hash值发生碰撞的概率比较小，这样就会使得数据在table数组中分布较均匀，查询速度也较快.

系统总是将新的Entry对象添加到bucketIndex处。如果bucketIndex处已经有了对象，那么新添加的Entry对象将指向原有的Entry对象，形成一条Entry链，但是若bucketIndex处没有Entry对象，也就是e==null,那么新添加的Entry对象指向null，也就不会产生Entry链了

Java-LinkedList

Posté le 2017-03-13

LinkedList和ArrayList一样实现了List接口, 只是ArrayList是List接口的大小可变数组的实现, LinkedList是List接口链表的实现.

LinkedList在插入和删除元素时更优于ArrayList, 而随机访问则是ArrayList优于LinkedList.

LinkedList实现了Deque接口, 为add, poll提供先进先出队列操作, 以及其他堆栈和双端队列操作.

所有操作都是按照双重链接列表的需要执行的. 在列表中编索引的操作将开头或结尾遍历列表

LinkedList和ArrayList一样也是不同步的.

LinkedList定义

AbstractSequentialList提供了List接口的主要实现. Deque一个线性的collection, 支持在两端插入和移除元素, 定义双端队列的操作.

LinkedList属性

LinkedList提供两个基本属性: size, header.

size表示LinkedList的大小

header表示链表表头, Entry 为节点对象

Entry是LinkedList的内部类, 定义了存储的元素. 该元素的前一个元素和后一个元素, 典型的双向链表定义

构造方法

LinkedList()构造一个空列表, 没有任何元素, 仅仅只把header节点的前一个元素, 后一个元素指向自身.

LinkedList(Collection<? extends E> c): 构造一个包含collection中元素的列表. 先调用LinkedList()构造一个空列表, 再调用addAll()方法将collection中的所有元素添加到列表中.

entry(int index)方法主要用来查找index位置的节点元素

增加元素方法

  • add(E e): 把置顶元素添加到列表结尾

  • addBefor方法

    addBefore方法构建一个新的节点newEntry, 然后修改他的前后引用

    LinkedList还提供类其他的增加元素的方法:

    • add(int index, E element)：在此列表中指定的位置插入指定的元素。
    • addAll(Collection<? extends E> c)：添加指定 collection 中的所有元素到此列表的结尾
    • addAll(int index, Collection<? extends E> c)：将指定 collection 中的所有元素从指定位置开始插入此列表。
    • AddFirst(E e): 将指定元素插入此列表的开头。
    • addLast(E e): 将指定元素添加到此列表的结尾。

移除元素方法

  • remove(Object o) 从列表中首次出现的元素移除.

  • remove(Entry e) : LinkedList所有移除方法的基础

    其他移除方法:

    • clear()： 从此列表中移除所有元素。
    • remove()：获取并移除此列表的头（第一个元素）
    • remove(int index)：移除此列表中指定位置处的元素
    • remove(Objec o)：从此列表中移除首次出现的指定元素（如果存在）
    • removeFirst()：移除并返回此列表的第一个元素
    • removeFirstOccurrence(Object o)：从此列表中移除第一次出现的指定元素（从头部到尾部遍历列表时）
    • removeLast()：移除并返回此列表的最后一个元素
    • removeLastOccurrence(Object o)：从此列表中移除最后一次出现的指定元素（从头部到尾部遍历列表时）

Java-ArrayList

Posté le 2017-03-13

ArrayList是实现List接口的动态数组. 实现所有可选列表操作, 并允许null在内的所有元素.

ArrayList的默认初始容量为10. 每次添加新的元素, ArrayList都会检查是否需要进行扩容操作.

扩容操作是把原来的数组拷贝到新的数组中.

如果已知具体的数据量, 在构造时应该指定一个初始容量, 可以减少数据拷贝次数.

在添加大量新元素前, 可以使用ensureCapacity操作来增加ArrayList实例的容量.

ArrayList实现不是同步的. 如果多个线程同时访问一个ArrayList实例, 其中至少一个线程从结构上修改了列表, 必须保持外部同步. 最好在创建时, 保证ArrayList时同步的.

ArrayList底层使用数组来保存数据.

transient是Java关键字, 变量修饰符. 当对象存储时, 它的值不需要维持.

Java的serialization提供了一个持久化对象实例的机制. 当持久化对象时, 可能有的对象数据成员, 不想用serialization机制保存它. 为了在一个特定的对象的一个域上关闭serialization, 可以在这个域前加上关键字transient. 当对象被序列化时, transient型变量的值不包括在序列号的表示中.

Object[] elementData就是ArrayList的容器. ArrayList的基本操作都是对elementData变量进行操作.

ArrayList提供三个构造函数.

  • ArrayList(): 默认构造器, 提供初始容量为10的空列表
  • ArrayList(int initialCapacity): 构造一个具有指定初始容量的空列表
  • ArrayList(Collection<? extends E> c): 构造一个包含指定collection的元素的列表, 这些元素时按collectin的迭代器返回的顺序排列的.

ArrayList 提供了add(E e), add(int index, E e), AddAll(Collection<? extends E> c), addAll(int index, Collection<? extends E> c), set(int index, E e); 这五个方法实现ArrayList增加元素.

  • add(E e): 把指定元素添加的列表尾部

    ensureCapacity()方法是对ArrayList集合进行扩容操作.

  • add(int index, E e): 把指定元素插入到列表的制定位置

    如果指定的数据集合需要大量插入操作, 推荐使用LinkedList

  • addAll(Collection<? extends E> c): 按照指定collection的迭代器所返回的元素顺序, 把该collection中的所有元素添加到列表的尾部.

    使用System.arraycopy()方法把c集合里面的数据复制到elementData数组中.

    public static void arraycopy(Object src, int srcPos, Object dest, int destPos, int length)方法的根本目的是进行数组元素的复制. 把原数组src从srcPos位置开始复制到dest数组中, 复制长度为length, 数据从dest的destPos位置开始粘贴.

  • addAll(int x, Collection<? extends E> c): 从指定位置开始, 把collection中的所有元素插入到列表中

  • set(int index, E e): 用指定元素替换列表中指定位置上的元素

ArrayList提供remove(int index), remove(Object o), removeRange(int formIndex, int toIndex), removeAll()四个方法进行元素删除

  • remove(int index): 从列表中删除指定位置上的元素

  • remove(Object o): 删除列表中首次出现的制定元素

    其中fastRemove()方法用于删除指定位置的元素

  • removeRange(int fromIndex, int toIndex): 删除列表中索引在formIndex(包含)和toIndex(不包含)之间的元素

  • removeAll(): 继承自AbstractCollectin方法

每次新增元素时都需要进行容量检查判断, 如果新增元素的个数超过ArrayList的容量, 就会进行扩容操作. 使用ensureCapacity方法进行扩容

每次扩容1.5倍既能满足性能需求, 也不会造成很大的内存消耗.

Java集合接口

Posté le 2017-03-10

Java类库中集合类的基本接口是Collection接口. 这个接口有两个基本方法:

add方法用于向集合添加元素, 如果添加元素改变了集合返回true, 否则就返回false.

iterator方法返回一个实现了Iterator接口的对象. 用这个迭代器对象依次访问集合中的元素.

Iterator接口有三个方法:

反复调用next方法可以逐个访问集合中的每个元素. 如果到了集合的末尾, next方法会抛出一个NoSuchElementException. 在调用next方法前要先调用hasNext方法.

使用for each循环遍历集合

编译器会把for each循环翻译为带有迭代器的循环.

Java迭代器的查找操作与位置变更是紧密相连的. 查找一个元素的唯一方法是调用next.

执行查找操作的同时, 迭代器的位置也向前移动.

应该认为Java迭代器是位于两个元素之间. 当调用next时, 迭代器会越过下一个元素, 并返回刚才越过的那个元素的引用.

Iterator接口的remove方法会删除上次调用next方法时返回的元素.

要删除指定位置上的元素, 必须要迭代器越过这个元素.

在调用remove之前必须要调用next, 否则会抛出一个IllegalStateException异常.
